# UML Designer 需求文档

> 按需求提出先后顺序整理，基于 `uml_design.md` 原始设计 + 开发过程中迭代的需求

## 第一阶段：项目初始化

### D01 - 项目代码生成
- **提出**: 会话开始
- **描述**: 基于 `uml_design.md` 设计文档生成完整的项目代码，前端 React+AntV X6，后端 FastAPI
- **状态**: ✅ 已完成

### D02 - 项目启动与访问
- **提出**: 会话开始
- **描述**: 前后端分离架构，前端端口 3000，后端端口 8000
- **状态**: ✅ 已完成
- **启动命令**:
  - 后端: `cd backend && python -m app.main`
  - 前端: `cd frontend && npx vite --port 3000`

## 第二阶段：基础编辑功能

### D03 - 添加类按钮
- **提出**: "画布顶端增加一个按钮，添加类"
- **描述**: 在工具栏添加"添加类"按钮，点击后在画布随机位置创建 UML 类节点
- **状态**: ✅ 已完成
- **额外**: 双击画布空白区域也可添加类

### D04 - 节点端口可见
- **提出**: "没有看到节点端口可以拖拽"
- **描述**: 每个类节点四边显示蓝色端口圆点，悬停高亮，从端口拖拽创建连线
- **状态**: ✅ 已完成
- **实现**: X6 ports 配置 + CSS 显隐

### D05 - 类节点位置不堆叠
- **提出**: "第一个类创建正常，后续的无法显示"
- **描述**: 多个类节点不应堆叠在同一坐标
- **状态**: ✅ 已完成
- **解决**: "添加类"按钮使用随机偏移坐标

### D06 - 连接属性修改实时更新
- **提出**: "连接属性修改关系类型后在画布上没有反应"
- **描述**: 在属性面板修改关系类型、多重性、角色名后，画布实时更新边的标签和线型
- **状态**: ✅ 已完成
- **解决**: 边同步逻辑改为同时处理新增和更新

---

## 第三阶段：文件操作

### D07 - 保存时选择位置和文件名
- **提出**: "保存uml设计的时候，请支持选择存储的地方和文件命名"
- **描述**: 
  - "保存"按钮改为下拉菜单（保存 / 另存为）
  - 另存为弹窗支持自定义文件名
  - 显示已有文件列表供参考
- **状态**: ✅ 已完成
- **存储路径**: `backend/uml_files/{filename}.uml`

### D08 - 打开文件支持浏览其他文件夹
- **提出**: "打开文件，请支持选择其他文件夹下面的文件"
- **描述**: 打开文件对话框支持目录导航，浏览任意文件夹下的 .uml 文件
- **状态**: ✅ 已完成
- **实现**: 后端 `/api/files/browse` 端点 + 前端目录导航 UI

---

## 第四阶段：LLM 优化与对比

### D09 - 优化设计弹窗输入需求
- **提出**: "优化设计的时候要弹窗等待用户需求输入，将用户需求和当前的设计文档组合发给llm优化"
- **描述**: 点击"优化设计"弹出需求输入对话框，用户输入优化指令后与 UML 数据一起发给 LLM
- **状态**: ✅ 已完成

### D010 - UML 优化对比画布切换
- **提出**: "uml优化对比，增加一个反转按钮，可以在画布上切换新旧版本"
- **描述**: 对比面板增加切换按钮，在画布上实时切换显示原始版和优化版
- **状态**: ✅ 已完成

### D011 - 评审意见 + dev_review.txt
- **提出**: "uml优化对比栏，要增加评审意见窗口，接受或者拒绝都要将评审信息和操作记录保存在本地"
- **描述**: 
  - 对比面板增加评审意见输入框
  - 接受/拒绝操作记录保存到 `backend/dev_review.txt`
  - 日志包含时间、操作类型、优化需求、版本信息、评审意见
- **状态**: ✅ 已完成

### D012 - 接受优化二次确认
- **提出**: "接受优化之后，跳出二次确认的窗口"
- **描述**: 点击"接受优化"后弹出确认对话框，确认后才应用优化结果
- **状态**: ✅ 已完成

### D013 - 拒绝后可继续优化
- **提出**: "拒绝优化之后，弹窗支持输入新的优化需求，点击确认继续优化"
- **描述**: 
  - 拒绝后弹出需求输入框，支持"继续优化"(发送新需求给LLM) / "放弃优化"(直接取消) / "取消"(关闭弹窗)
  - 评审完成后按钮置灰
- **状态**: ✅ 已完成

### D014 - 重新优化后按钮状态重置
- **提出**: "再次执行优化设计后，接受和拒绝按钮仍然置灰"
- **描述**: 新优化结果到达后，DiffViewer 状态自动重置，按钮恢复可用
- **状态**: ✅ 已完成

### D015 - 操作日志带上优化请求内容
- **提出**: "操作日志上带上优化的请求内容"
- **描述**: `dev_review.txt` 评审日志中增加"优化需求"字段，记录用户输入的原始优化指令
- **状态**: ✅ 已完成

## 第五阶段：网格系统

### D016 - 网格开关修复
- **提出**: "网格开关目前没有效果"
- **描述**: 修正网格显隐切换逻辑，使用 X6 原生 `showGrid()`/`hideGrid()` API
- **状态**: ✅ 已完成

### D017 - 网格颜色和粗细配置
- **提出**: "网格配置要支持线条颜色和粗细的调整"
- **描述**: 
  - 网格设置弹窗：大小下拉、颜色选择器+文本输入、粗细滑块(1-5)
  - 所有设置实时生效并保存到 .uml 文件
- **状态**: ✅ 已完成

## 第六阶段：自动化流水线

### D018 - 流水线动态状态显示
- **提出**: "流水线启动之后，流水线界面要动态展示当前执行的动作"
- **描述**: 
  - 实时状态栏显示当前阶段和状态（⏳/✅/❌ 图标 + 动作描述）
  - Steps 列表运行中阶段显示旋转加载图标
  - 进度条动画
- **状态**: ✅ 已完成

### D019 - 流水线停止按钮
- **提出**: "增加流水线停止的按钮"
- **描述**: 
  - 运行中/等待确认时显示红色"停止"按钮
  - WebSocket 发送 `{action: "stop"}` 终止后端流水线
- **状态**: ✅ 已完成

### D020 - 流水线第一步弹出优化需求
- **提出**: "第一步uml优化，需要跳出窗口提示是否需要增加需求进行优化，然后跳转到uml确认面板"
- **描述**: 
  - Stage1 开始时弹出需求输入窗口
  - "提交优化"→ 带需求调用 LLM → 完成后跳转对比面板
  - "跳过"→ 跳过 Stage1+Stage2，直接进入 Stage3 代码生成
- **状态**: ✅ 已完成
